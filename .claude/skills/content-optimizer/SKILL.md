# Content Optimizer Skill

## What This Skill Does

Optimizes content for maximum engagement across platforms:
- **Platform-specific formatting** - Character limits, markdown, emojis
- **Tone adaptation** - Professional, casual, technical, friendly
- **Hook optimization** - Attention-grabbing openings
- **Structure optimization** - Scannable, readable, mobile-friendly
- **Hashtag strategy** - Relevant, trending, not spammy
- **CTA optimization** - Clear, compelling calls-to-action
- **Engagement tactics** - Questions, value-first, controversy balance
- **A/B variant generation** - Multiple versions for testing

## When This Skill Activates

This skill automatically activates for requests like:
- "Optimize this post for Reddit"
- "Create a LinkedIn version of this content"
- "Turn this into a Twitter thread"
- "Make this more engaging for HackerNews"
- "Adapt this for Discord community"
- "Generate social media posts from this article"
- "Create A/B test variants for this content"

## Platform-Specific Guidelines

### Reddit

**Tone:** Casual, helpful, anti-marketing
**Format:** Markdown, bullet points, code blocks
**Hook:** Problem-focused, relatable, data-driven
**Length:** 500-1500 words for valuable posts
**Emoji usage:** Minimal (data/metrics only: 📊 📈)
**Hashtags:** Never use hashtags
**CTA:** Soft, value-first ("Happy to share more if useful")

**Structure:**
```
Hook: Relatable problem or surprising data

Body:
• Clear problem statement
• What you tried and failed
• What actually worked (with data)
• Specific implementation steps

CTA: Offer to help, answer questions
```

**Avoid:**
- Self-promotion without value
- Marketing speak
- Excessive emojis
- Links in main post (comments OK)

**Best practices:**
- Lead with data/results
- Show vulnerability (what didn't work)
- Provide specific, actionable advice
- Engage in comments genuinely

### LinkedIn

**Tone:** Professional but authentic, thought leadership
**Format:** Short paragraphs, line breaks for scanability
**Hook:** Bold statement, question, or data point
**Length:** 1300-1500 chars (3-4 paragraphs) or long-form 1500+
**Emoji usage:** Professional emojis for structure (✅ 📊 💡)
**Hashtags:** 3-5 relevant hashtags at end
**CTA:** Clear professional action

**Structure:**
```
Hook: Bold opening line (standalone)

Problem paragraph (2-3 lines)

Solution paragraph (2-3 lines)

Result paragraph with data (2-3 lines)

CTA: What should reader do next?

#Hashtag1 #Hashtag2 #Hashtag3
```

**Avoid:**
- Walls of text
- Humble bragging
- Vague insights
- Excessive hashtags (>5)

**Best practices:**
- Use line breaks generously
- Lead with value
- Include specific numbers
- Tag relevant people/companies
- Post during business hours (Tue-Thu 8-10 AM)

### Twitter/X

**Tone:** Punchy, opinionated, conversational
**Format:** Thread for longer content, single for quick takes
**Hook:** Controversial or counterintuitive statement
**Length:** 280 chars per tweet, 8-12 tweets max for threads
**Emoji usage:** Moderate, for emphasis and personality
**Hashtags:** 1-2 max, highly relevant
**CTA:** Retweet, follow, or click link

**Single Tweet Structure:**
```
Hook/insight (first 140 chars - visible without "show more")

Supporting point or data

CTA or hashtag
```

**Thread Structure:**
```
1/ Hook tweet with thread indicator "🧵"

2/ Context or problem setup

3-9/ Main points (one per tweet)

10/ Summary and CTA

(Number each tweet: 1/, 2/, etc.)
```

**Avoid:**
- Overly formal tone
- Generic insights
- Too many hashtags
- Long threads without value

**Best practices:**
- Front-load the insight
- Use contrarian takes
- Include visuals when possible
- Reply to your own thread with link
- Post 1-3 PM or 8-10 PM

### HackerNews

**Tone:** Technical, skeptical, intellectually curious
**Format:** Plain text, technical depth, no markdown
**Hook:** Technical insight or architectural decision
**Length:** Detailed explanation (500-2000 words)
**Emoji usage:** None (kills credibility)
**Hashtags:** Never
**CTA:** Invite technical discussion

**Structure:**
```
Technical hook or surprising decision

Deep technical context:
- Architecture choices
- Performance considerations
- Trade-offs evaluated
- Implementation details

Specific numbers and benchmarks

Open questions or areas for improvement
```

**Avoid:**
- Marketing language
- Buzzwords
- Emojis
- Superficial explanations
- Conclusions without data

**Best practices:**
- Show your work (code, benchmarks, trade-offs)
- Admit unknowns and limitations
- Invite criticism and alternatives
- Respond to all technical questions
- Post early morning (6-8 AM PT) or late night

### Discord

**Tone:** Friendly, casual, community-focused
**Format:** Short messages, code blocks, embeds
**Hook:** Direct question or helpful announcement
**Length:** 50-200 chars per message (short bursts)
**Emoji usage:** Heavy, expressive, server-specific
**Hashtags:** Never
**CTA:** Engage in thread, react with emoji

**Structure:**
```
Quick hook or question

Main point (1-2 sentences)

Optional: Code block, screenshot, or link

Invite response: "Thoughts?" "Anyone else?"
```

**Avoid:**
- Long walls of text (break into messages)
- Cross-posting spam
- Ignoring server culture
- Marketing without permission

**Best practices:**
- Use server-specific emojis
- Break long content into multiple messages
- Use threads for extended discussions
- React to messages to show engagement
- Match community energy level

## Optimization Instructions

When this skill activates, follow these steps:

### 1. Identify Target Platform

**Ask if not specified:**
- "Which platform should I optimize this for?"
- Default: Reddit if discussing problems/solutions
- Default: LinkedIn if professional/business context
- Default: Twitter if quick insight/hot take

### 2. Analyze Source Content

**Extract key elements:**
- Main message/insight
- Supporting data/evidence
- Call-to-action
- Target audience
- Tone/voice

**Identify optimization opportunities:**
- Hook weakness
- Structure issues
- Tone misalignment
- Length problems
- Missing elements

### 3. Apply Platform Guidelines

**Transform content:**
- Rewrite hook for platform
- Adjust tone appropriately
- Restructure for platform format
- Optimize length (add/cut)
- Add platform-specific elements

**Platform checklist:**
- [ ] Tone matches platform culture
- [ ] Format follows platform norms
- [ ] Hook is attention-grabbing
- [ ] Structure is scannable
- [ ] Length is appropriate
- [ ] Emoji usage is correct
- [ ] Hashtags (if applicable) are relevant
- [ ] CTA is clear and platform-appropriate

### 4. Enhance Engagement Factors

**Hook optimization:**
- Use data/numbers
- Start with surprising insight
- Ask compelling question
- Make bold/controversial claim (if defensible)
- Lead with transformation/result

**Structure optimization:**
- Break into scannable chunks
- Use bullet points or numbered lists
- Add white space (line breaks)
- Highlight key data with formatting
- Create rhythm and flow

**Value optimization:**
- Lead with value, not context
- Be specific, not generic
- Include actionable takeaways
- Provide proof/data
- Show, don't just tell

### 5. Generate A/B Variants

**Create 2-3 versions with different approaches:**

**Variant A: Data-driven hook**
- Lead with numbers/results
- Focus on proof and metrics
- Appeal to analytical audience

**Variant B: Problem-focused hook**
- Lead with relatable problem
- Focus on pain points
- Appeal to people seeking solutions

**Variant C: Contrarian hook**
- Challenge common belief
- Focus on unique perspective
- Appeal to thought leaders

**Label variants clearly:**
```
VARIANT A (Data-driven):
[Optimized content]

VARIANT B (Problem-focused):
[Optimized content]

VARIANT C (Contrarian):
[Optimized content]
```

### 6. Provide Strategic Guidance

**Best posting times:**
- Reddit: 6-8 AM ET (before work browsing)
- LinkedIn: Tue-Thu 8-10 AM (business hours)
- Twitter: 1-3 PM or 8-10 PM (commute/evening)
- HackerNews: 6-8 AM PT (morning reading)
- Discord: Anytime (depends on community)

**Engagement tactics:**
- Respond to all comments within first hour
- Have 2-3 follow-up points ready
- Engage with related posts/threads
- Cross-promote subtly across platforms

**A/B testing:**
- Test different hooks same content
- Track engagement by variant
- Double down on what works
- Iterate based on data

## Quality Standards

**Authenticity:**
- Maintain original voice while adapting tone
- Don't lose key message in optimization
- Be genuine, not manipulative

**Value-first:**
- Every post must provide clear value
- No clickbait without substance
- Respect audience intelligence

**Platform alignment:**
- Follow platform culture religiously
- Don't force content where it doesn't fit
- Adapt substantially, don't just tweak

**Engagement optimization:**
- Optimize for genuine engagement, not just virality
- Encourage meaningful discussion
- Build community, not just followers
