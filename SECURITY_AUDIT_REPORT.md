#### 1. Hardcoded Database Password
**Risk:** High
**Location:** `docker-compose.yml`
**Issue:** Database password `healthdata_password` is hardcoded in docker-compose.yml and exposed to all services.

**Recommendation:**
- Use environment variable from `.env` file (not committed)
- Use Docker secrets or external secrets manager
- Rotate password before production deployment

#### 2. Demo JWT Secret
**Risk:** High
**Location:** `docker-compose.yml`
**Issue:** JWT secret `mySecretKeyForJWTShouldBeAtLeast256BitsLongForHS256Algorithm123456` is shared across all services in plain text.

**Recommendation:**
- Generate cryptographically secure secret: `openssl rand -base64 64`
- Store in secrets manager (Vault, AWS Secrets Manager, etc.)
- Use different secrets per environment

#### 3. All Service Ports Exposed
**Risk:** Medium
**Location:** `docker-compose.yml`
**Issue:** 18 service ports are exposed to 0.0.0.0, including:
- PostgreSQL (5435)
- Redis (6380)
- Kafka (9094)
- All microservice ports (8080-8107)

**Recommendation:**
- Only expose gateway port (8080) publicly
- Internal services should not be directly accessible
- Use Docker network isolation

#### 4. CORS Configuration for Localhost Only
**Risk:** Medium
**Location:** `gateway-service/application.yml`
**Issue:** CORS allows localhost origins (4200, 4201, 4202)

**Recommendation:**
- Configure production domains before deployment
- Use environment variable: `CORS_ORIGINS`
