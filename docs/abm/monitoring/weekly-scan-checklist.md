# Weekly Scan Checklist

**Time estimate:** 15 minutes per week
**When:** Monday morning (or first work day of the week)
**Output:** Update [trigger-log.md](./trigger-log.md) with any new triggers found

---

## Sources to Check

### 1. Google Alerts (3 minutes)

Review the weekly digest email for each active target.

- [ ] Open Google Alerts digest
- [ ] Scan for: quality reporting, Star ratings, leadership changes, acquisitions, VBC contracts
- [ ] Log any relevant hits in trigger-log.md

**Setup:** Create a Google Alert for each active target's organization name. Set to weekly digest.

### 2. CMS.gov (3 minutes)

Check for new data releases relevant to active targets.

- [ ] [Star Ratings](https://www.cms.gov/medicare/health-drug-plans/part-c-d-performance-data) -- annual release in October, mid-year previews
- [ ] [ACO Performance Files](https://data.cms.gov/medicare-shared-savings-program/accountable-care-organizations-performance) -- quarterly updates
- [ ] [Quality Data Releases](https://data.cms.gov/) -- new datasets or updates
- [ ] [FQHC UDS Data](https://data.hrsa.gov/tools/data-reporting/program-data) -- annual release

### 3. LinkedIn (5 minutes)

Check each target's decision-maker profiles.

- [ ] Scan for new posts by decision makers (quality, VBC, technology topics)
- [ ] Check for job title changes or new hires at target orgs
- [ ] Look for organization announcements (partnerships, expansions, awards)
- [ ] Note any engagement opportunities (comment-worthy posts)

### 4. Google News (2 minutes)

Quick search for each active target.

- [ ] Search: `"[Org Name]" quality OR acquisition OR value-based OR contract`
- [ ] Search: `"[Org Name]" HEDIS OR Stars OR NCQA OR accreditation`
- [ ] Log any relevant results in trigger-log.md

### 5. NCQA / HEDIS (2 minutes)

Check for industry-level changes that affect targets.

- [ ] [NCQA News](https://www.ncqa.org/news/) -- accreditation changes, new standards
- [ ] [HEDIS Updates](https://www.ncqa.org/hedis/) -- new measure year announcements, measure changes
- [ ] Note any changes that create urgency for specific targets

---

## After the Scan

1. **Log triggers:** Add any new findings to [trigger-log.md](./trigger-log.md)
2. **Check response windows:** If a trigger fires, initiate the matching outreach sequence within the response window (see trigger-log.md for windows)
3. **Update tracker:** Update [targets/_tracker.md](../targets/_tracker.md) with new trigger dates and status changes
4. **Update dossiers:** Add new intelligence to the relevant target dossier in `targets/[org-name].md`
