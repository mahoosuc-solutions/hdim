# BSL and Commercial Summary (Executive)

**Date:** 2026-03-07  
**Product:** HealthData-in-Motion (HDIM)  
**Company:** Grateful House Incorporated

## 1) Licensing Position in Plain English

HDIM is **source-available** under Business Source License 1.1 (BSL 1.1), not
an OSI open-source license.

- Non-production use (evaluation, development, testing) is allowed under BSL.
- Production use is limited by the LICENSE Additional Use Grant.
- Commercial production rights are provided through paid commercial agreements.

Primary source:
- [LICENSE](/mnt/wdblack/dev/projects/hdim-master/LICENSE)

## 2) What Commercial Licensing Adds

Commercial agreements grant rights not fully covered by repository BSL terms,
including:

- Production use for customer operations (internal use)
- Optional affiliate rights
- Optional hosted/service-provider rights
- Support, service levels, and compliance addenda

Primary source:
- [COMMERCIAL_LICENSE_TERMS_OUTLINE.md](/mnt/wdblack/dev/projects/hdim-master/docs/commercial/COMMERCIAL_LICENSE_TERMS_OUTLINE.md)

## 3) Rights and Package Options

### Rights Model

- `A1` Internal Use Only
- `A2` Affiliate Use
- `A3` Service Provider Rights

### Deployment Model

- `B1` Customer-Hosted
- `B2` Shared Responsibility
- `B3` Licensor-Hosted (optional/future)

### Support Model

- `C1` Standard
- `C2` Priority
- `C3` Enterprise

### Package Mapping

- `Starter`: A1 + B1/B2 + C1
- `Growth`: A1/A2 + B1/B2 + C2
- `Enterprise`: A2/A3 + B1/B2/B3 + C3

## 4) Pricing Grouping Framework

Default approach is quote-based, using one primary grouping and optional add-ons.

Primary grouping options:

- Covered lives bands (recommended default)
- Tenant/entity bands
- Volume/event bands

Common add-ons:

- Implementation bundle
- Compliance bundle
- Advanced support bundle
- Commercial rights bundle

Primary source:
- [PRICING_PAGE_CONTENT.md](/mnt/wdblack/dev/projects/hdim-master/docs/commercial/PRICING_PAGE_CONTENT.md)

## 5) Commercial Flow (How a Deal Is Structured)

1. Qualify rights needed (A1/A2/A3)
2. Confirm deployment model (B1/B2/B3)
3. Select support tier (C1/C2/C3)
4. Select pricing grouping + add-ons
5. Issue quote + order form
6. Attach compliance docs if needed (e.g., BAA/security addendum)

## 6) Fast FAQ

### Is HDIM open source?

No. HDIM is source-available under BSL 1.1.

### When is a commercial license required?

For production use not granted by the repository LICENSE, including hosted or
service-provider scenarios.

### Is public list pricing available?

Not currently. Pricing is quote-based while package calibration is finalized.
