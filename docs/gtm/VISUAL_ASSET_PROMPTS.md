# HDIM Visual Asset Prompts for Gemini Pro 3

## Quick Reference Guide

This document contains ready-to-use prompts for generating visual assets with Gemini Pro 3.

## Brand Guidelines

**Primary Color**: Deep Blue `#0066CC`
**Accent Color**: Warm Teal `#00A5B5`
**Background**: White/Light Gray
**Style**: Modern, warm, human-centered, professional
**Mood**: Hopeful, trustworthy, innovative, caring

## 3. The 5-Minute Story Visual

### Split Screen Composition (1920x1080)
```
Create a powerful split-screen image representing "code that saves lives."

LEFT SIDE (50%):
A computer terminal/code editor with a dark background (like VS Code dark theme).
The commit message is clearly visible:
"fix(hipaa): Reduce PHI cache TTL to ≤5min for HIPAA compliance"
Date visible: December 27, 2025, 10:31 PM
The code screen should look authentic, with syntax highlighting.

RIGHT SIDE (50%):
A warm, touching family moment. A grandmother (could be Maria) with grandchildren,
or a father (could be James) playing with his kids. The image should convey
"this is what we're protecting."

TRANSITION:
The split between the two sides should feel connected - perhaps flowing lines of
light/data that transition from the code side to the family side, suggesting the
code protects the people.

Overall mood: The juxtaposition of cold technology and warm humanity, connected by purpose.
```

### Square Version for Social (1080x1080)
```
Same concept as split screen, but in square format for Instagram/social. Code on top
third, family image on bottom two-thirds, with visual connection between them.
```

## 4. Technical/Architecture Visuals

### Microservices Architecture (1200x800)
```
Create an elegant, modern technical diagram showing a microservices architecture.

Layout: 27 hexagonal nodes arranged in an organic, connected pattern. The nodes should
feel organized but not rigidly grid-like - more like a living network.

Each node has:
- A simple icon inside (heart, shield, database, gear, etc.)
- Subtle glow effect
- Connection lines to related nodes

Key nodes to highlight (larger/brighter):
- FHIR Service (database icon)
- CQL Engine (code brackets icon)
- Care Gap Service (heart with alert icon)
- Quality Measure Service (chart icon)
- Patient Service (person icon)

Connection lines: Flowing, subtle, suggesting data movement. Not harsh straight lines -
gentle curves that feel organic.

Color palette: Deep blue (#0066CC) for primary nodes, teal (#00A5B5) for accent nodes,
white/light gray background, subtle gradients.

Style: Clean, professional, modern SaaS aesthetic. Should feel trustworthy and organized.
NOT cluttered or overwhelming.
```

### Data Flow Animation Frames (Set of 5)
```
Create 5 sequential frames showing data flowing through the microservices network.

Frame 1: Data enters from EHR source (left edge) - highlighted incoming connection
Frame 2: Data passes through FHIR Service - FHIR node pulses
Frame 3: Data flows to CQL Engine and Care Gap Service - dual nodes pulse
Frame 4: Results aggregate at Quality Measure Service - central node pulses
Frame 5: Insights flow to dashboard output (right edge) - completion state

Each frame should show subtle "flow" animation along the connection lines.
Use color gradient (blue to teal) to indicate data movement direction.
```

## Quick Generation Checklist

When generating each asset, verify:
- [ ] Colors match brand guidelines (#0066CC, #00A5B5)
- [ ] Style is consistent (modern, warm, professional)
- [ ] No stock photo "feel" on portraits
- [ ] Text is readable and well-positioned
- [ ] Dimensions match specifications
- [ ] Mood aligns with brand (hopeful, trustworthy, innovative)
- [ ] Healthcare context is authentic, not cliché

## Output Formats Needed

For each asset, generate:
1. **Web**: High-quality PNG or JPG
2. **Print** (if applicable): High-resolution PDF
3. **Social**: Optimized versions for each platform
