# Release Scope: v2.7.1-rc1

**Date:** 2026-02-15  
**Previous:** v2.7.0

## Why This Release Exists

- Establish a repeatable Release Candidate lane for demo readiness, media capture, and release promotion.
- Lock Nx + MCP tooling so agent workflows and local gates are deterministic.
- Produce the minimum “public demo” proof set aligned to validated market differentiators (predictive care gaps + AI context + FHIR-native platform).

## Included In Scope

- Tooling + workflow (local):
  - Local CI runner: `scripts/ci/local-ci.sh` (quick/pr/demo/release modes).
  - MCP validation gate: `npm run test:mcp`.
  - Nx/MCP version alignment and wrapper-based `.mcp.json` configuration.
- Release lane documentation:
  - RC lane definition: `docs/RELEASE_CANDIDATE_LANE.md`.
  - Demo readiness gate: `docs/DEMO_READINESS_GATE.md`.
  - Capture checklist: `docs/media/CAPTURE_CHECKLIST.md`.
- Release artifacts for this RC:
  - `docs/releases/v2.7.1-rc1/*` (notes, matrix, known issues, scope, deployment checklist).

## Explicitly Out Of Scope

- Any net-new product surface area not required for the demo readiness gate.
- “Phase 3” real-time financial ROI dashboard claims unless verified as demo-able (treat as preview/roadmap).
- Broad refactors or dependency upgrades unrelated to stability, demo determinism, or security.

## Demo Readiness Acceptance Criteria

- Demo stack boots from scratch (`docker-compose.demo.yml`).
- Seeding is deterministic (`./scripts/seed-all-demo-data.sh`).
- System validation passes (`./validate-system.sh`).
- Research-driven “hero set” flows are screenshot/video capture ready (see `docs/media/CAPTURE_CHECKLIST.md`):
  - Predictive care gaps surfaced on dashboard (30-60 day look-ahead) and drilldown to action.
  - Care gaps execution loop (prioritize, view detail, take an action or document next step).
  - FHIR-native proof point via gateway (`/fhir/metadata` plus at least one read path).
  - Auditability proof point (audit log viewer and at least one detail view/export).

## Release Validation Evidence

- Local CI gates:
  - `./scripts/ci/local-ci.sh quick`
  - `./scripts/ci/local-ci.sh pr`
  - `./scripts/ci/local-ci.sh demo`
- MCP gate:
  - `npm run test:mcp`
- Release validation reports:
  - `docs/releases/v2.7.1-rc1/validation/*` (generated by scripts under `scripts/release-validation/`)

## Rollback Plan

- If RC validation fails:
  - Revert only the scoped commits on `release/v2.7.1-rc1` and re-run `./scripts/ci/local-ci.sh pr`.
  - If needed, reset RC branch to `v2.7.0` baseline and re-cherry-pick only required fixes.
- If a deployment fails (demo/prod):
  - Roll back to the previous tagged release (`v2.7.0`) images/manifests and re-run health checks.
