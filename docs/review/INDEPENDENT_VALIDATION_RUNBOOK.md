# Independent Validation Runbook

Date: 2026-02-25

## Goal

Enable third-party reviewers to independently validate architecture and release claims.

## Preconditions

- Docker installed and running
- Node.js + npm available
- Java/Gradle available for backend checks
- GitHub CLI authenticated for release API checks

## Step 1: Generate static evidence

```bash
bash scripts/review/generate-technical-review-pack.sh
```

Expected output:
- `docs/review/evidence/technical-review-evidence-<timestamp>.md`

Preferred (automated checks + evidence):

```bash
bash scripts/review/run-technical-review-checks.sh
```

Expected output:
- `docs/review/evidence/technical-review-checks-<timestamp>.md`
- `docs/review/evidence/technical-review-evidence-<timestamp>.md`

## Step 2: Verify release/tag consistency

```bash
git tag --list | sort -V
gh release list --repo webemo-aaron/hdim --limit 50
gh api repos/webemo-aaron/hdim/releases/tags/v2.7.0
gh api repos/webemo-aaron/hdim/releases/tags/v2.7.1-rc2
```

Pass criteria:
- v2.7 tags exist
- v2.7 release objects exist and are retrievable

## Step 3: Architecture/document alignment spot-check

Check consistency between:
- `docs/architecture/SYSTEM_ARCHITECTURE.md`
- `yc-application-v2/TECHNICAL_ARCHITECTURE.md`
- generated evidence metrics

Record mismatches (counts, service inventory, version statements).

## Step 4: Runtime validation (demo profile)

```bash
docker compose -f docker-compose.demo.yml up -d
./scripts/seed-all-demo-data.sh
./validate-system.sh
```

Pass criteria:
- services healthy
- seeding succeeds
- validator returns successful checks

## Step 5: Frontend smoke

```bash
npm run e2e:clinical-portal:smoke
```

Pass criteria:
- smoke suite passes on current branch

## Step 6: Reviewer report

Create report sections:
1. Environment and commit hash
2. What passed
3. What failed
4. Discrepancies and confidence level
5. Go/no-go recommendation

## Notes

- This runbook is validation-first and does not assume claims are true.
- Evidence generated by commands should be treated as primary source.
- To include runtime validation in the machine-generated report, run:

```bash
bash scripts/review/run-technical-review-checks.sh --with-runtime
```
