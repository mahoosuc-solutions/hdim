# Runbook Launch Checklist (Next 7 Days)

Audience: Founder, Solutions Architect, Engineering Lead, Customer Success Lead
Objective: Put the MA/ACO/IPA runbook system into active execution within one week.

## Day 1: Assign Owners and Operating Cadence

- [ ] Assign one owner for each runbook:
  - [ ] MA runbook owner
  - [ ] ACO runbook owner
  - [ ] IPA runbook owner
- [ ] Assign template/SOP librarian owner
- [ ] Set weekly runbook review meeting (30 minutes)
- [ ] Confirm escalation path for blocked decisions

Deliverables:
- Owner list
- Standing calendar invite
- `operations/OWNER_ROSTER_TEMPLATE.md` completed

## Day 2: Baseline Current In-Flight Customer(s)

- [ ] Select active customer opportunity/pilot and map to customer type
- [ ] Copy the relevant runbook into working execution notes
- [ ] Capture current status for gates G1/G2/G3
- [ ] Record current risks in risk register format

Deliverables:
- Customer-type mapping
- Initial gate status snapshot
- Initial risk register entries
- `operations/GATE_STATUS_SNAPSHOT_TEMPLATE.md` completed

## Day 3: Enforce Week-1 Deployment Decision Process

- [ ] Confirm deployment options presented (hosted/customer-cloud/on-prem/air-gapped)
- [ ] Confirm required stakeholders invited (clinical, IT, security, procurement)
- [ ] Schedule 90-minute Week-1 architecture workshop
- [ ] Pre-distribute gate evidence template

Deliverables:
- Workshop invite
- Decision packet

## Day 4: Gate Evidence Collection Dry Run

- [ ] Complete one dry-run `GATE_EVIDENCE_TEMPLATE.md`
- [ ] Validate required evidence is obtainable from current process
- [ ] Identify missing evidence producers and assign owners

Deliverables:
- Completed dry-run gate evidence doc
- Gap list with owners

## Day 5: SOP Adoption for Cross-Functional Team

- [ ] Review SOP checklists with each function:
  - [ ] Discovery and qualification
  - [ ] Integration readiness
  - [ ] Security readiness
  - [ ] Pilot kickoff readiness
  - [ ] Weekly steering
- [ ] Confirm handoff points between Sales -> SA -> Engineering -> CS

Deliverables:
- SOP adoption sign-off
- Handoff matrix
- `operations/HANDOFF_MATRIX_TEMPLATE.md` completed

## Day 6: KPI and Steering Operating Loop Setup

- [ ] Confirm KPI definition for active customer:
  - [ ] Engagement rate (7-day)
  - [ ] Comprehension confirmation
  - [ ] First response time
  - [ ] Escalation closure time
- [ ] Create first `WEEKLY_STEERING_UPDATE_TEMPLATE.md`
- [ ] Validate decision log and action log owner

Deliverables:
- Week 1 steering template populated
- `operations/ACTION_LOG_TEMPLATE.md` initiated

## Day 7: Go-Live Runbook Readiness Review

- [ ] Review each customer-type runbook for completeness
- [ ] Confirm hard-gate policy is understood (no kickoff without evidence)
- [ ] Confirm all required templates are discoverable by team
- [ ] Publish “ready for execution” status note

Deliverables:
- Runbook readiness memo
- Updated risk register
- `operations/RUNBOOK_READINESS_MEMO_TEMPLATE.md` completed

## Definition of Done (End of Day 7)

- [ ] At least one live customer is fully mapped to MA/ACO/IPA runbook flow
- [ ] Week-1 deployment decision gate process is active
- [ ] Gate evidence template is in active use
- [ ] Weekly steering loop is running
- [ ] Clear owners exist for runbook maintenance and escalation
