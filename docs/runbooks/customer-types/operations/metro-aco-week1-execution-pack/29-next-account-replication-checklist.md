# Next Account Replication Checklist

Purpose: replicate this execution system for the next MA/ACO/IPA account.

## Replication Trigger

Start this checklist when current account reaches:

- [ ] Week 1 complete and G1 closed
- [ ] Stable operating rhythm established

## Replication Steps

1. Duplicate the execution pack folder for new account.
- [ ] New folder created under `docs/runbooks/customer-types/operations/`

2. Replace account-specific references.
- [ ] Customer name
- [ ] Customer type
- [ ] Pilot dates
- [ ] Owner roster

3. Reset operational states.
- [ ] Action log statuses reset to Open/Not Started
- [ ] Risk register reset to account-specific initial risks
- [ ] Decision log reset to Pending

4. Validate communication templates.
- [ ] Invite email subject/body reviewed
- [ ] Pre-read list updated to correct references
- [ ] Follow-up template reviewed

5. Publish readiness for new account.
- [ ] New pack README updated
- [ ] Command-center checklist ready
- [ ] Day 1 send set prepared

## QA Check

- [ ] No references to prior customer remain
- [ ] All dates and owners are account-correct
- [ ] Gate model remains unchanged (G1/G2/G3)
