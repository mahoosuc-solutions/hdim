# Stakeholder Messaging Map

Prepared: 2026-02-23
Purpose: Ensure one coherent story for clinical, IT/security, and procurement audiences.

## 1) Core Message by Stakeholder

### Clinical Sponsor (CMO, VP Care Management, Medical Director)

Primary message:

1. This pilot is designed to improve post-discharge engagement and patient understanding in 90 days.
2. AI is used to translate complex discharge context into patient-friendly guidance, with escalation to clinicians when needed.

Proof points to emphasize:

1. 200-500 discharge cohort with weekly KPI tracking.
2. Focused metrics: engagement, comprehension, response speed, escalation closure.
3. Phase 1 excludes risky write-back workflows until validated.

### Operations / Care Transition Team

Primary message:

1. The platform supports practical day-to-day workflows, not just analytics.
2. Escalation and notification paths are operationally usable from day one.

Proof points to emphasize:

1. Secure in-platform escalation queue.
2. HIPAA-secure email alert option.
3. Weekly governance rhythm and issue-triage cadence.

### Security / Compliance / Privacy

Primary message:

1. The architecture enforces minimum-necessary PHI handling and zero-retention AI posture.
2. Deployment can match internal policy via hosted, customer-cloud, or on-prem/air-gapped options.

Proof points to emphasize:

1. Tenant isolation and audited access patterns.
2. No customer-data model training.
3. Deployment-specific shared responsibility boundaries.

### IT / Enterprise Architecture

Primary message:

1. The solution is deployable in multiple topologies without redesign.
2. Integration and operational controls are explicit and phase-gated.

Proof points to emphasize:

1. Supported topologies: HDIM-hosted, customer-cloud, on-prem/air-gapped.
2. Week-1 decision gate locks timeline and build sequence.
3. MyChart write-back deferred to controlled Phase 2.

### Procurement / Finance

Primary message:

1. Commercials are transparent, with a hosted base estimate and clear deployment-specific adders.
2. Pilot structure reduces risk before broader scale commitment.

Proof points to emphasize:

1. Base hosted total: $277,000-$383,000.
2. Explicit model adders and assumptions.
3. Milestone-based payment structure.

## 2) Message Hierarchy (Use in Every Conversation)

1. Outcomes first: what improves and how we measure it.
2. Trust second: security and PHI controls.
3. Delivery third: deployment model and timeline.
4. Commercials fourth: base + adders, with assumptions.
5. Ask last: Week-1 workshop and deployment decision gate.

## 3) Consistency Rules

1. Always state the base estimate assumes HDIM-hosted deployment.
2. Always include deployment alternatives when discussing architecture or cost.
3. Never imply MyChart write-back is in Phase 1 scope.
4. Always end with explicit next-step commitment request.
