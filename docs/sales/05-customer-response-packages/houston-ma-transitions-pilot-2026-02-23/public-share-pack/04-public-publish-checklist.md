# Public Publish Checklist

## Content Safety

- [ ] No customer-specific names, emails, or identifiers
- [ ] No internal-only operational logs or owner rosters
- [ ] No unapproved legal/commercial terms
- [ ] No internal escalation paths or risk-by-person details

## Message Consistency

- [ ] Outcome claims match pilot scope
- [ ] Phase 1 vs Phase 2 boundaries are clear
- [ ] Deployment models are described consistently
- [ ] Pricing shown only at approved public level

## Distribution Readiness

- [ ] Final files exported to PDF/web format as needed
- [ ] Internal stakeholder approval captured
- [ ] External links tested
- [ ] Version/date stamp confirmed
