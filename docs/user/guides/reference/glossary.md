# Glossary & Definitions

Platform and clinical terms explained.

## Platform Terms

**API**: Application Programming Interface. Technical connection between systems enabling data exchange.

**Audit Log**: Record of all user actions and data changes for compliance and security tracking.

**Dashboard**: Main view showing patient panels, alerts, metrics, and key information.

**EHR**: Electronic Health Record. Source system of clinical and patient data.

**Integration**: Connection between platform and EHR or other systems enabling data sync.

**MRN**: Medical Record Number. Unique patient identifier in healthcare system.

**Portal**: Web application where users access platform.

**Query**: Request to retrieve specific data from system.

**Role**: User type determining permissions and features available (physician, care manager, admin, etc.).

**Sync/Synchronization**: Automatic update of data from EHR to platform.

## Clinical Terms

**Allergy**: Adverse reaction to medication or substance. Must be documented and verified.

**Chronic Disease**: Long-term disease requiring ongoing management (diabetes, hypertension, etc.).

**Clinical Alert**: Flag indicating patient safety issue requiring prompt attention.

**Condition**: Patient diagnosis or medical problem.

**Contraindication**: Reason not to prescribe medication or perform procedure.

**Diagnosis**: Identification of patient disease or condition.

**Encounter**: Patient visit or interaction (office visit, phone call, message).

**Evaluation**: Assessment of patient condition or testing outcome.

**Finding**: Clinical observation or test result.

**Health Maintenance**: Preventive care and screenings to maintain health.

**Intervention**: Treatment action taken (medication, referral, procedure).

**Medication Reconciliation**: Process of verifying patient's accurate medication list.

**Observation**: Clinical finding from examination or monitoring.

**Procedure**: Medical intervention or surgery.

**Referral**: Request to specialist for evaluation and care.

**Status**: Current state of patient condition or order.

## Quality Measure Terms

**Baseline**: Starting point for measuring improvement.

**Benchmark**: Standard or comparison point for performance.

**Care Gap**: Recommended care not yet provided to patient.

**Closure**: Completion of care gap by providing recommended service.

**Cohort**: Group of patients sharing specific characteristic.

**Denominator**: Number of patients eligible for measure.

**Exclusion**: Criteria removing patient from measure (contraindication, patient refused, etc.).

**HEDIS**: Healthcare Effectiveness Data and Information Set. Standardized quality measures.

**Measure**: Standard for assessing quality of care.

**Metric**: Specific measurement or data point.

**Numerator**: Number of eligible patients who received recommended care.

**Outcome**: Result of care or treatment.

**Performance**: How well organization meets quality standards.

**Population**: Group of patients being measured.

**Quality Score**: Percentage of recommended care provided.

## Task and Workflow Terms

**Assignment**: Assigning task or patient to specific team member.

**Batch**: Group of items processed together.

**Completion**: Task or process finished successfully.

**Deadline**: Due date for task completion.

**Delegation**: Assigning responsibility to another person.

**Due Date**: Target date for task completion.

**Escalation**: Moving issue to higher authority for resolution.

**Notification**: Alert to user about event or assignment.

**Priority**: Urgency level (high, medium, low).

**Status**: Current state (pending, in-progress, completed, archived).

**Task**: Specific action item assigned to person.

**Workflow**: Series of steps for completing process.

## Reporting Terms

**Aggregate**: Combine data from multiple sources or patients.

**Column**: Vertical data field in report.

**Data Point**: Single measurement or value.

**Export**: Save or transfer report data to another format.

**Filter**: Criteria to narrow dataset (date range, patient type, etc.).

**Format**: Presentation style (PDF, Excel, CSV, etc.).

**Query**: Request for specific data.

**Report**: Organized presentation of data and analysis.

**Row**: Horizontal record in report.

**Schedule**: Automated regular report generation.

**Sort**: Arrange data in specific order.

**Trend**: Pattern in data over time.

## Data and Technical Terms

**Archive**: Store data without active access.

**Backup**: Copy of data for recovery.

**Cache**: Temporary stored data for faster access.

**Data Migration**: Transfer of data from one system to another.

**Database**: Organized collection of data.

**Encryption**: Coding data for security.

**Field**: Single data element.

**Firewall**: Security barrier controlling network access.

**HIPAA**: Health Insurance Portability and Accountability Act. Privacy and security law.

**Metadata**: Information about data (who, when, what changed).

**Query**: Request to retrieve data.

**Record**: Complete data for one patient or item.

**Restore**: Recover data from backup.

**Security**: Protection of data from unauthorized access.

**Timeout**: Session ending due to inactivity.

**Validation**: Verification that data meets standards.

## User Role Terms

**Administrator**: User managing system, users, and settings.

**Care Manager**: User coordinating patient care and outreach.

**Clinician**: Healthcare provider delivering direct care.

**Executive**: Leadership user viewing organization-level data.

**Physician**: Doctor providing patient diagnosis and treatment.

**Supervisor**: Manager overseeing team.

**Support Staff**: Users providing administrative support.

## Access and Security Terms

**Access Control**: Restriction of system use to authorized users.

**Authentication**: Verification of user identity (login, MFA).

**Authorization**: Permission to perform specific actions.

**Credentials**: Username and password.

**MFA**: Multi-Factor Authentication. Extra security layer requiring code/device.

**Password**: Secret code for account access.

**Permission**: Allowance to perform specific action.

**Role-Based Access**: Permissions determined by user role.

**Session**: Period of time user is logged in.

**Two-Factor Authentication**: Security requiring password and additional code.

## Troubleshooting Terms

**Bug**: Software error or malfunction.

**Cache Clear**: Removing stored browser data.

**Debug**: Process of finding and fixing errors.

**Error Code**: Number identifying specific problem.

**Error Message**: Text describing problem.

**Logs**: Record of system events and errors.

**Refresh**: Reload page to get latest data.

**Timeout**: Session or request ending due to inactivity.

**Troubleshoot**: Process of identifying and solving problems.
