# Security & Privacy Guide

Protecting patient data and system security.

## Access Security

### Passwords
- **Create strong password**: 12+ characters, mix of cases, numbers, symbols
- **Don't share password**: Keep credentials private always
- **Change regularly**: Update password every 90 days
- **Don't reuse passwords**: Use unique password for this system
- **Don't write down**: Use password manager if needed
- **Never email passwords**: IT never asks for password
- **Recognize phishing**: Be suspicious of unexpected emails asking for credentials

### Multi-Factor Authentication (MFA)
- **Enable MFA**: Strongest security practice
- **Keep device secure**: Don't share phone/hardware token
- **Don't share codes**: Code is single-use, don't tell anyone
- **Save backup codes**: Store securely if MFA codes provided
- **Update if device lost**: Contact IT if lose phone/token
- **Use authenticator app**: More secure than SMS when available

### Login
- **Log out when done**: Always logout, especially shared devices
- **Lock screen**: Lock computer when stepping away
- **Check URL**: Verify correct website (not phishing)
- **Trust warnings**: If browser warns about site, don't proceed
- **Use secure network**: Avoid public WiFi for sensitive work
- **Never use "Remember Me"**: On shared or public computers

## Data Handling

### Patient Information
- **Limit view**: Only access patient data you need to see
- **Don't browse**: Don't look up records without clinical reason
- **Protect on screen**: Shield from view of others
- **Print carefully**: Only print what necessary
- **Don't email**: Avoid emailing patient data if possible
- **Use secure email**: If must email, use encrypted email
- **Destroy copies**: Shred or properly delete printed records
- **Don't photograph**: Don't take photos of screens with PHI

### Messaging
- **Don't include unnecessary PHI**: Minimum information needed
- **Use secure messaging**: In-system messaging more secure than email
- **Verify recipient**: Confirm messaging right person
- **Don't discuss sensitive**: Avoid sensitive info in messages
- **Be professional**: Maintain appropriate tone
- **Assume permanent**: Don't write anything you regret long-term

### Reporting
- **Limit report sharing**: Only share with authorized users
- **Use secure methods**: Encrypted email or secure portal
- **Verify recipient**: Confirm authorized to receive
- **De-identify if possible**: Remove identifying information if not needed
- **Restrict distribution**: Limit number of copies
- **Destroy when done**: Delete when no longer needed

## HIPAA Compliance

### Protected Health Information (PHI)
- **Know what's protected**: Any individually identifiable health info
- **Understand your role**: What data can you access?
- **Follow authorization**: Only use data for authorized purposes
- **Minimum necessary**: Only access/use minimum needed
- **Document access**: Your access is audited and tracked
- **Report breaches**: Immediately report any suspicious activity

### Privacy Rules
- **Consent**: Patient has rights to their data
- **Amendments**: Patients can request record corrections
- **Disclosures**: Track who gets patient data
- **Use limitations**: Only use data for authorized reasons
- **Accounting**: Patient can request list of who accessed records
- **Privacy practices**: Understand organization's privacy notice

### Security Rules
- **Encryption**: Sensitive data transmitted securely
- **Unique user ID**: System tracks who does what
- **Audit controls**: All access logged and reviewed
- **Integrity**: Data hasn't been altered
- **Transmission**: Data in transit protected
- **Disposal**: Old data properly disposed

## Device and Network Security

### Computer
- **Keep updated**: Install all security updates promptly
- **Use antivirus**: Maintain updated antivirus software
- **Avoid USB drives**: Don't use unauthorized USB storage
- **Don't disable security**: Keep firewall and protection active
- **Scan for malware**: Regular malware scans
- **Physical security**: Lock computer when away
- **Don't lend device**: Personal device security risk

### Mobile Device
- **Use strong PIN**: 6+ digits, not easy pattern
- **Keep updated**: Install all security updates
- **Install apps safely**: Only from official app store
- **Use VPN**: On public WiFi, use VPN
- **Don't jailbreak**: Keep OS security controls intact
- **Encrypt data**: Enable device encryption
- **Remote wipe**: Enable remote wipe in case loss
- **Report loss**: Contact IT immediately if lost/stolen

### Network
- **Avoid public WiFi**: For sensitive work, use secure network
- **Use VPN**: When on public WiFi, use organizational VPN
- **Don't trust networks**: Don't assume open network safe
- **Verify network**: Confirm network name with IT
- **Use mobile hotspot**: Phone data often more secure than WiFi
- **Disable auto-connect**: Don't auto-connect to networks
- **Clear history**: Remove trusted networks periodically

## Incident Response

### Security Incident
- **Don't panic**: Immediate response is more important
- **Isolate if possible**: Disconnect from network if malware
- **Contact IT immediately**: Report all security concerns
- **Document details**: Note what happened and when
- **Preserve evidence**: Don't try to clean up yourself
- **Follow instructions**: Follow IT guidance for containment
- **Incident report**: May need to file formal report

### Suspicious Activity
- **Unknown login**: Someone accessing your account?
- **Missing data**: Records disappear or change?
- **Unusual messages**: Messages you didn't send?
- **Unauthorized access**: Access to patients not assigned?
- **Report immediately**: Don't wait, contact IT urgently
- **Change password**: May need password reset
- **Monitor account**: Watch for further unusual activity

### Potential Breach
- **Accidental disclosure**: Sent data to wrong person?
- **Lost device**: Lost phone or laptop?
- **Public exposure**: Data visible to unauthorized person?
- **Report immediately**: HIPAA requires prompt reporting
- **Contain issue**: Stop further exposure if possible
- **Notify supervisor**: Inform management immediately
- **Documentation**: Document everything about incident

## Training and Awareness

### Staying Informed
- **Attend training**: Complete required security training
- **Read updates**: Review security alerts from IT
- **Ask questions**: Clarify security expectations
- **Report gaps**: If something doesn't seem secure
- **Share knowledge**: Help colleagues stay secure
- **Review policies**: Understand security policies
- **Test knowledge**: Take security assessment if available

### Phishing and Social Engineering
- **Recognize phishing**: Suspicious emails asking for credentials
- **Be skeptical**: Be suspicious of unexpected requests
- **Verify requests**: Call person if unsure about request
- **Don't click links**: In suspicious emails, type URL instead
- **Don't download**: Be careful about email attachments
- **Check sender**: Verify actual email address (not display name)
- **Report phishing**: Forward suspicious emails to IT

## Acceptable Use

### Appropriate Use
- **Work purposes**: Use system for work only
- **Patient care**: Access data only for assigned patient care
- **Professional**: Maintain professional conduct always
- **Respect privacy**: Protect patient and colleague privacy
- **Follow policies**: Adhere to security policies
- **Use wisely**: Don't abuse system privileges
- **Model behavior**: Set good example for others

### Inappropriate Use
- **Personal use**: Don't use system for personal business
- **Unauthorized access**: Don't access data without reason
- **Browse records**: Don't look up records out of curiosity
- **Share credentials**: Don't share login with anyone
- **Tampering**: Don't attempt to bypass security
- **Disclosure**: Don't share patient data inappropriately
- **Gaming system**: Don't try to circumvent security measures

## Compliance Checklist

### Daily
- ✅ Lock screen when away
- ✅ Logout when done
- ✅ Verify no one watching you type password
- ✅ Don't discuss PHI publicly
- ✅ Report suspicious activity

### Weekly
- ✅ Review access log (who accessed your records?)
- ✅ Verify authorized staff have access
- ✅ Check for unusual messages
- ✅ Confirm mobile device secure

### Monthly
- ✅ Change password if due
- ✅ Review security settings
- ✅ Clear browser cache
- ✅ Verify backups running
- ✅ Update antivirus

### Annually
- ✅ Attend security training
- ✅ Review policies and procedures
- ✅ Assess device security
- ✅ Verify HIPAA understanding
- ✅ Certification if applicable
