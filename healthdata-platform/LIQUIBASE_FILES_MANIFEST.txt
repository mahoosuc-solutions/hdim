================================================================================
QUICK START
================================================================================

1. Verify PostgreSQL is running and database 'healthdata' exists

2. Update application.yml with database credentials:
   spring:
     datasource:
       url: jdbc:postgresql://localhost:5432/healthdata
       username: postgres
       password: password
     liquibase:
       change-log: classpath:db/changelog/db.changelog-master.xml

3. Run the application:
   ./gradlew bootRun

4. Migrations run automatically on startup

5. Verify with SQL:
   \dn                    # List schemas
   SELECT * FROM information_schema.tables
   WHERE table_schema NOT IN ('pg_catalog', 'information_schema');

================================================================================
IMPORTANT NOTES
================================================================================

1. Migration Order:
   - Migrations MUST run in order (001 → 009)
   - Dependencies are clearly defined
   - Liquibase handles ordering automatically

2. Rollback Support:
   - Every changeset includes rollback instructions
   - Full rollback capability for all changes
   - Can rollback by count, date, or individual changeset

3. Multi-Tenancy:
   - All tables include tenant_id column
   - Indexes support tenant isolation
   - All queries should filter by tenant_id

4. Performance:
   - 70+ indexes for query optimization
   - GIN indexes on JSONB columns
   - Composite indexes for common queries
   - Partial indexes for filtered conditions

5. Audit Trail:
   - Comprehensive audit logging
   - User action tracking
   - Field-level change history
   - HIPAA compliance support
